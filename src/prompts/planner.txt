3. Establish dependencies using the `depends_on` array. A tester usually depends on a coder.
4. Keep scopes narrow. Narrow scopes equal fewer hallucinations.

OUTPUT FORMAT:
You must respond with ONLY valid, raw JSON representing the TaskPlan. 
Do NOT wrap the JSON in markdown blocks (e.g., no ```json).
Do NOT include any conversational text before or after the JSON.

  "subtasks": [
